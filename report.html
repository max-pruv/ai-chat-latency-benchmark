Methodology

Each assistant was driven on its live customer site. Latency = wall-clock from message sent to full response (network/WebSocket/DOM timing per vendor; Meta AI screenshot-approx). Two question pools were run: Shopping Assistant (discovery questions that ask for a product recommendation) and Support (shipping FAQ + returns/policy). Answer quality was scored by a blind 3-judge LLM panel on anonymized stores.

Caveats: different stores/domains (treat as directional); Gorgias ran on its own NouriVida demo (sales-tuned, best-case); Ada's bot was down on both test days; Meta AI's underlying model isn't verifiable client-side (front-end is Zendesk's "Virtual Assistant").

Cold vs warm sessions. Each widget persists its conversation in its own cross-origin storage (gorgias.chat, siena.cx, …), which a page script can't clear, and the automation can't drive a true incognito window — so re-runs reuse a "warm" session that slightly inflates latency (measured: DigitalGenius next-day answer 12.7s cold vs 14.3s warm). We therefore test each vendor cold-first (one clean pass, no re-runs in the same window). The recurring monthly run must spin up a fresh browser profile/container per execution — that's the only reliable way to guarantee a cold session for every vendor.
