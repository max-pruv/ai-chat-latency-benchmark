AI Shopping Assistant Benchmark — Latency & "Did it finish the job?"

Same ~10-turn shopping-discovery conversation run on each store's live on-site AI assistant. We measure two things: response latency and the success rate — the share of turns the AI handles itself, without spontaneously handing off to a human. We never ask for a human; an unprompted handover on a normal question is the failure we're measuring.

Read it straight (with the fair-comparison nuance): The clearest separator isn't speed — it's whether the assistant finishes the shopping job or bails to a human. Spiffy.ai (Supergoop) completed 100% of a 10-turn discovery and never handed off. The two Gorgias stores both escalate early — Madura hands to a human agent at ~turn 5, Masderm transfers to a human + email-gate at turn 3. That's merchant-configuration-driven (login/email gating, agent routing), not a fixed engine limit — a sales-tuned setup can keep the AI in the driver's seat — but as configured today, these two stores don't complete the shopping journey on their own.

Method & caveats

• One standardized shopping script per store — open ask → needs/context → budget → compare two → objection → secondary need → bundle → best-seller → add to cart → total/shipping — adapted to each catalog so it's apples-to-apples. We never request a human; any handover is therefore unprompted.

• "Spiffy.ai" on Supergoop is the vendor actually serving Supergoop's on-site assistant (the user originally referred to it as "Envive"; the live widget is Spiffy.ai — commerce-api.spiffy.ai, spiffy-* components).

• Latency is directional — different catalogs, single test windows, and some session/timing noise. Treat cross-store numbers as indicative, and read the success column alongside speed.

• Gorgias is "us"; the two stores tested (Madura, Masderm) are real customer storefronts with their own escalation settings — both escalate to a human early in a shopping chat by configuration (login/email gating, agent routing), which a sales-tuned setup can change.

• Coverage in progress. The 2nd store for each competitor and standardized re-tests of the legacy single-store vendors are being added; this page updates as they land.
