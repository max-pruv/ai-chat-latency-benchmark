How the Gorgias AI Agent stacks up against every rival

How this benchmark works

A head-to-head test of the Gorgias AI Agent against every rival's live on-site AI — measured on identical footing.

The test

The same scripted, multi-turn conversations are run against each store's live on-site assistant — apple-to-apple across vendors. Each store gets 5 themed conversations per mode (~7 turns each): five Shopping themes (everyday value, gift, problem-solver, comparison, beginner) and five Support themes (tracking, returns, damaged item, order changes, policy).

Cold & private

Every conversation runs in a fresh, isolated browser session — no warm history, a genuine cold start. We never ask to speak to a human; an unprompted handover on a normal question is exactly the failure we're measuring.

What we measure

  • Latency — true end-to-end: from the instant the shopper sends a message to the instant the full reply finishes rendering. Only the AI's own turns are timed; once a human takes over, those turns are excluded.
  • Self-service rate — the share of turns the AI handled itself, with no spontaneous handover to a human.
  • Answer quality — graded on Gorgias's own evaluation rubrics (below).

Sample window

Runs accumulate: each new run adds dated conversations and keeps the old ones. The date picker narrows the sample to a window (default: last 30 days), so every score reflects exactly the runs you've selected.

Best AI agent — how the ranking works

The two jobs an on-site assistant is hired for are ranked by a single composite that weighs the two things that matter most: success (does it finish the job without bailing to a human) and speed (latency to the full answer). Composite = 60% success + 40% speed · speed scored 100 at ≤3s, 0 at ≥22s. Averaged per vendor across its stores; respects the date window. Ranking is gated on measurement coverage — a vendor needs enough cleanly-timed turns to be ranked on speed; those with too few are listed separately.

Method & caveats

  • Apples-to-apples script — the same open-ask → needs → budget → compare → objection → bundle → best-seller → add-to-cart → total/shipping flow, adapted to each catalog. We never request a human, so any handover is unprompted.
  • Once a human takes over, we stop — the remaining scripted turns aren't sent to a live agent; they count as not-self-served so the success denominator stays honest.
  • Latency is directional — different catalogs, single windows, some timing noise. Read success and coverage alongside speed, not a single number.
  • Gorgias is "us" — tested on real customer storefronts; behavior varies by each merchant's config.
  • "Envive" = Spiffy.ai (same company, rebranded) — treated as one vendor.
  • Coverage is honest, not uniform — only Gorgias drives fully headless; competitor widgets are headed-only and some have hard ceilings (offline widgets, handover-only). Stores with no timed AI turn are left unscored, never invented.
